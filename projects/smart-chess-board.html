Smart Chess Board

Embedded sensing with load cells and signal processing for chess piece detection.

Context

This project explored whether a grid of load cells beneath a chessboard could reliably detect piece positions without camera vision. The focus was on low-cost sensing, embedded processing, and firmware that could stream consistent board state updates.

The work served as a hardware-software integration exercise, emphasizing sensor selection, mechanical packaging, and digital filtering to clean up noisy weight signals.

Approach

The board used four load cells arranged in a wheatstone configuration per square to capture force distribution, feeding an HX711 ADC into a microcontroller for sampling and serial output. Firmware averaged multiple reads per square and applied thresholding to map summed forces to occupancy states.

Bench tests evaluated detection accuracy across piece types and positions, iterating on thresholds and debounce timing to minimize false positives from adjacent squares.

Implementation & Tools

  • HX711 ADC modules and load cells mounted under a custom chessboard frame
  • Microcontroller firmware in C for sampling, averaging, and threshold classification
  • Python serial interface for logging and quick visualization during calibration
  • 3D-printed mounts and foam isolation to reduce cross-talk between squares

Key Results

After calibration, the board consistently distinguished occupied from unoccupied squares for standard pieces with minimal cross-talk, though heavier pieces required higher thresholds that slowed response slightly.

Latency from contact to reliable detection was typically under 300 ms when averaging 10–15 samples, balancing responsiveness against noise suppression.

Analysis & Insights

Load-cell arrays can work for board-state detection if mechanical isolation and per-square calibration are prioritized; most noise issues traced back to flex in the board rather than electronics. Threshold tuning benefited from grouping pieces by weight instead of a single global value.

Future iterations would explore per-square auto-zeroing and dynamic gain adjustments to maintain sensitivity as the board flexes with repeated use.

What This Demonstrates

  • Embedded sensing design with load cells and ADC integration
  • Firmware development for low-latency signal processing and classification
  • Iterative calibration and testing to tune thresholds and debounce behavior
  • Mechanical packaging to reduce sensor cross-talk in dense arrays
